Accessible Math Writing Checklist

Designed for screen reader–friendly LaTeX → HTML conversion (MathML / MathJax) with NVDA-style navigation.

I. Structural Clarity

✅ 1. Introduce variables in prose when possible

Prefer prose scaffolding:

For all real numbers \(a\) and \(b\), we have …

Over compact symbolic binders:

\forall a,b \in \mathbb{R} ( ... )

✅ 2. Avoid chained binder-parenthesis patterns

Avoid:

\forall x(\exists y(\forall z(...)))

Prefer:

\forall x\, (\exists y\, (\forall z\, (...)))

✅ 3. Do not use parentheses as punctuation

Avoid:

(\forall X \subseteq A)(X=\emptyset \vee ...)

Prefer:

\forall X \subseteq A:\ X = \emptyset \text{ or } ...

✅ 4. Make logical scope explicit

Use explicit separators: \,, :, \text{ such that }, \text{ and }, \text{ or }. Avoid relying on adjacency for meaning.

II. Math Expression Hygiene

✅ 5. Avoid ambiguous juxtaposition

Avoid:

xy

Prefer explicit intent:

x\,y
x \cdot y

✅ 6. Always group exponents and subscripts

Avoid:

x^12
x_i+1

Prefer:

x^{12}
x_{i+1}

✅ 7. Prefer explicit fractions

Avoid:

a/b

Prefer:

\frac{a}{b}

✅ 8. Use semantic delimiters for absolute value

Prefer:

\lvert x \rvert

Over:

|x|

✅ 9. Don’t chain meaning with adjacency parentheses

Avoid:

(...)(...)(...)

Prefer explicit connectors:

... \land ...
... \text{ and } ...

III. Logical Structure

✅ 10. Use colon in set-builder notation

\{ x \in A : P(x) \}

✅ 11. Use words for connectives when clarity matters

Instead of heavy symbolic chaining (\land, \lor), consider:

\text{ and }
\text{ or }

✅ 12. Break long formulas across lines when possible

A multi-line display block is often easier to follow than a single long line, especially for speech.

IV. Quantifier Discipline

✅ 13. Separate quantifiers from scope

Avoid:

\forall x(...)

Prefer:

\forall x\, (...)

This prevents screen readers from interpreting x( as “times open paren”.

✅ 14. Avoid stacking quantifiers without pacing

Avoid:

\forall x \forall y \exists z (...)

Prefer prose or add spacing:

\forall x\, \forall y\, \exists z\, (...)

V. Macros and Environments

✅ 15. Be cautious with compact custom macros

If you define shortcuts like \newcommand{\R}{\mathbb{R}}, ensure your conversion pipeline expands them appropriately, or expand them in source when needed.

✅ 16. Don’t rely on theorem environments for navigation

Make theorems and definitions navigable (e.g., convert them into headings in HTML output) so screen reader users can jump between them quickly.

VI. Cognitive Load Check

After writing a paragraph or equation, ask:

“If I close my eyes and hear this read aloud, does it feel calm or compressed?”

If compressed, add explicit structure (spacing, punctuation, words) or move logic into prose.

VII. Golden Principle

Make implicit structure explicit. PDF tolerates ambiguity; speech does not.
